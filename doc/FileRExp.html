Class FileRegularExpr

Derived from IRegularExpression
Derived by: none

#include <FileRExp.h>

Description

Class to compare text with (UNIX-file-style) regular expressions.

Note:Depending on the operating system this comparison is either case-sensitive (*X) or just case-preserving (DOS)!

The following characters have a special meaning:

  • *: matches any number of any characters
  • ?: matches any single character
  • [<match>]: matches the characters specified in match.
  • [^<match>]: matches the characters not specified in match.
  • [!<match>]: matches the characters not specified in match.

<match> ::= <char><match> | <range><match> | <character-class><match> | {}
<range> ::= <low>-<high>
<character-class> ::= [:<class>:]
<class> ::= alnum | alpha | cntrl | digit | space | graph | lower | print | punct | space | upper

Note: To include the character square bracket ([) in the match, it must be the first character; similar to the caret (^), wich must not be the first character to get included.

Note that (for the sake of compatibility with other systems) the charcter classes "ascii" and "blank" are not supported.

Examples are: [[:alnum:]]* for all words starting with a letter or a number or [^A-Z]?? for all three character long words starting without an (capital) letter.

Note: The pRegExp-parameter is stored as is (and not copied); so care yourself for all the neccessary cleanup-stuff!

Use the matches-method of the parent (IRegularExpression to check if the objects matches some data.

bool compare (const char* pAktRegExp, const char* pCompare)

Checks if the passed text matches the passed regular expression.

Both passed parameter must not be NULL-pointers and the regular expression must be in a valid syntax (this can be checked with the checkIntegrity-method).
