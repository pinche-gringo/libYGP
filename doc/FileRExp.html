Class FileRegularExpr

Derived from: None

Header-file: FileRExp.h

Description

Class to compare text with (UNIX-file-style) regular expressions.

The following characters have a special meaning:

  • *: matches any number of any characters
  • ?: matches any single character
  • [<match>]: matches the characters specified in match
  • [^<match>]: matches the characters not specified in match.

<match> ::= | <char><match> | <region><match> | {}
<region> ::= <low>-<high>

Examples are: [a-zA-Z0-9]* for all words starting with a letter or a number or [^aeiuo]?? for all three letter long words starting with no vocal).

Note: The pRegExp-parameter is stored as is (and not copied); so care yourself for all the neccessary cleanup-stuff!

Matching

bool matches (const char* pCompare)

Checks if the passed text matches the stored regular expression. This method returns true, if yes and false if not.

Both the stored regular expression and the passed parameter must not be NULL-pointers and the regular expression must be in a valid syntax (this can be checked with the checkIntegrity-method).

Syntax-check

int checkIntegrity () const
Checks the syntax of the regular expression. If everything is OK, 0 is returned; 1 if there's the start of a region without a proper end, 2 if a list in the region has no end-character and 3 if a region is empty.
