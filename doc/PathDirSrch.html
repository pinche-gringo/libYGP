Class PathDirectorySearch

Derived from: DirectorySearch

Header-file: PathDirSrch.h

Description

Class to search for files in directory listed in a path. This search can be restricted to files matching certain name-criterias or by attributes.

The found (and matching) files are retrieved by objects of type dirEntry.

Find functions

int find (const std::string& path, const std::string& search, dirEntry& result, unsigned long attribs = FILE_NORMAL) - Searches for files specified by search in the passed paths; if at least one is found, the result is placed into result. The parameter attribs specify which kind of files to find. Possible values are FILE_NORMAL for "ordinary" files, FILE_READONLY for files which can not be changed (no w-flag in UNIX or read-only attribute set in DOS/WINDOZE),FILE_DIRECTORY searches for directories and FILE_HIDDEN for files not visible (files starting with a dot (.) in UNIX or with hidden- or system-flag set in DOS/WINDOZE).

This call returns 0, if a file was found, errno otherwise.

int find (const std::string& search, dirEntry& result, unsigned long attribs = FILE_NORMAL) - Searches for files specified by search in the paths which must have been set before; if at least one is found, the result is placed into result. The parameter attribs specify which kind of files to find. Possible values are FILE_NORMAL for "ordinary" files, FILE_READONLY for files which can not be changed (no w-flag in UNIX or read-only attribute set in DOS/WINDOZE),FILE_DIRECTORY searches for directories and FILE_HIDDEN for files not visible (files starting with a dot (.) in UNIX or with hidden- or system-flag set in DOS/WINDOZE).

This call returns 0, if a file was found, errno otherwise.

virtual int find (dirEntry& result, unsigned long attribs = FILE_NORMAL) - Searches for the first file matching the input specified by the constructor or the set-methods. The first found file is placed into result, the parameter attribs specify which kind of files to find. Possible values are FILE_NORMAL for "ordinary" files, FILE_READONLY for files which can not be changed (no w-flag in UNIX or read-only attribute set in DOS/WINDOZE), FILE_DIRECTORY searches for directories and FILE_HIDDEN for files not visible (files starting with a dot (.) in UNIX or with hidden- or system-flag set in DOS/WINDOZE).

This call returns 0, if a file was found, errno otherwise.

virtual int find () - Returns the next matching file according to parameters specified in early find-calls. If a file was found 0 is returned, errno otherwise.
